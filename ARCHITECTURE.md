## 🎯 核心数据流

### 1. 用户注册/登录流程
```
用户 ──► /login ──► SecondMe OAuth ──► /api/auth/callback 
                                        │
                                        ▼
                              保存用户信息+accessToken到DB
                                        │
                                        ▼
                              A2A Matcher自动匹配Top 50用户
                                        │
                                        ▼
                              创建A2AConversation记录(pending)
```

### 2. A2A 对话执行流程
```
定时任务或手动触发
      │
      ▼
读取pending状态的A2A记录
      │
      ▼
获取两个用户的accessToken
      │
      ▼
A2A Engine调用SecondMe Chat API
      │
      ├──► 用户A的AI分身发言
      │         │
      │         ▼
      │    中继服务器转发给用户B的AI分身
      │         │
      │         ▼
      │    用户B的AI分身回复
      │         │
      │         ▼
      └──► 循环3-5轮
                │
                ▼
         分析对话生成契合度报告
                │
                ▼
         保存到DB + 更新Dashboard推荐
```

### 3. 实时聊天流程
```
用户A ──► 发送消息 ──► /api/chat/send
                            │
                            ▼
                     保存到ChatMessage表
                            │
                            ▼
                     SSE推送给用户B (如果在线)
                            │
                            ▼
                     用户B实时收到消息
```

## 📝 关键设计决策

1. **A2A 中继模式** - 服务器中转而非P2P，便于记录对话和生成报告
2. **SSE 实时推送** - 比WebSocket更轻量，适合单向消息推送
3. **Prisma ORM** - 类型安全，自动迁移
4. **Standalone 部署** - 单容器部署，适合Railway
